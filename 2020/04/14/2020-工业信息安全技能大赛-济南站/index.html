工控协议数据分析（ARP 协议）

重点看 ARP 报文，可以发现 192.168.1.55 这台设备多次发出异常 ARP

192.168.1.0/24 中的 192.168.1.0 是网络地址，正常不应该被配置为主机地址或网关地址

所以这里的异常点是 PLC / 工控设备可能配置了错误的网关、通信目标或网络参数，导致它不断尝试 ARP 解析 192.168.1.0 从而产生大量告警日志

Reverse

某嵌入式设备固件升级包（VxWorks 固件）

题目给出的是一个名为 UpgradePackage.bin 的固件升级包

首先需要理解固件升级包的常见结构：大多数升级包包含一个明文头部（记录设备型号、版本等信息），后面跟着压缩或加密的固件本体

用十六进制编辑器或 binwalk 查看文件，能发现开头有可读字符串：140-NOE-771-01 和 Quantum Ethernet Executive firmware Ver. 6.40

这告诉我们设备型号和固件版本，同时也提示固件可能是 VxWorks 操作系统

VxWorks 是一种嵌入式实时操作系统，广泛应用于工业控制、网络设备中，其固件镜像通常包含符号表用于调试

继续分析升级包，需要找到真正的固件数据在哪里

偏移 0x385 处存在 zlib 压缩数据，用 Python 直接从 0x385 开始 zlib 解压

在解压后的固件里可以看到 VxWorks 痕迹

这证实了是 VxWorks 固件，并且带有 standalone symbol table（独立符号表）

standalone symbol table 是 VxWorks 系统在编译链接时附加到镜像末尾的一个表，包含了所有函数和全局变量的名称、地址和类型信息

符号表在固件分析中极为重要，因为它提供了函数名称到地址的映射，让我们可以快速定位目标函数，而不必进行繁琐的逆向工程

VxWorks 的 standalone 符号表结构由两部分组成：一连串的 16 字节符号条目，以及可能紧跟其后的一个 4 字节条目数量字段

每个符号条目为 16 字节，大端序（Big-Endian，即高位字节在前），其 C 结构体定义为

这里引入了 “虚拟地址 (VA)” 和 “加载基址 (load bias)” 的概念

固件文件本身存放的数据是文件偏移（file offset），当固件被加载到内存运行时，会被放置到某个起始地址，该起始地址就是加载基址

虚拟地址 = 文件偏移 + 加载基址

要分析符号表，必须先知道加载基址，因为表中存储的都是 VA

如何得到加载基址？可以通过已知字符串的文件偏移和它在符号表中的 name_va 来计算

固件里有很多以 \x00 结尾的 ASCII 字符串，比如

这些字符串都有明确的文件偏移

同时固件里有很多 4 字节大端整数，看起来像指针。我们可以做一件事

对每个可能的指针值 V，尝试看 V - base 是否刚好落到某个字符串起始偏移

如果某个 base 能让大量指针都指向合法字符串起点，那这个 base 就很可能是加载基址

大量 4 字节值满足 指针值 - 0x10000 = 某个字符串的文件偏移

接下来需要定位符号表本身

定位方法：搜索固件中可能表示符号条目数量的 4 字节大端整数

因为条目数量通常不会太小（一般上千），也不应过大（不超过几万），所以设定合理范围如 0x1000 到 0x10000

对于每个候选数量 n，往前推 n * 16 字节作为符号表起始地址

然后验证起始处的前 20 个条目和最后 20 个条目是否合法

验证条件：条目的第一个字段 unk 为 0，type 字段属于已知的有效类型集合（例如 0x100、0x200、0x300 等），且 name_va 转换到文件偏移后指向一个以 \x00 结尾的可打印字符串（长度小于 200）

如果 80% 以上条目合法，就可以确认找到了符号表

此方法在不知道确切的符号表偏移时非常有效，具有通用性

应用到本题，最终定位到符号表起始偏移为 0x301E60，符号数量字段偏移 0x3293B0，值为 0x00002755

现在有了符号表，就可以查找目标函数 Absolute__C14SequenceNumber

该字符串的文件偏移已在前面找到（0x26634C），其虚拟地址为 0x27634C

遍历符号表，找到 name_va 等于 0x27634C 的条目

该条目文件偏移为 0x302010，解析得到

value_va 即目标函数的加载地址 0x11FD38

转换回文件偏移：函数文件偏移 = 0x11FD38 - 0x10000 = 0x10FD38

到此已经拿到了两个关键数据：符号条目数量 0x2755，目标函数的文件偏移 0x10FD38

因此 flag 为 flag{000027550010FD38}
