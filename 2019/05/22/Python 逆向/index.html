Python

Python 逆向是指对 Python 编写的程序（包括脚本、.pyc 字节码、打包的可执行文件等）进行分析，以理解其逻辑、提取算法、恢复源码或寻找漏洞

Python 程序的编译与运行机制

源码 .py 被编译为 字节码 .pyc，存储在 __pycache__ 目录（Python 3.2+）

.pyc 文件结构：

  • 前 4 字节：magic number（标识 Python 版本，如 Python 3.8 为 550D）

  • 后 4 或 12 字节：时间戳 + 源码大小（可选）

  • 剩余部分：marshal 序列化后的 code object

逆向的核心就是：获取 .pyc → 反编译为近似源码或直接分析字节码

PyInstaller 将 Python 依赖模块的字节码 .pyc（去掉 8~16 字节头部）进行 zlib 压缩，然后打包成一个 PYZ 存档，其物理结构如下

TOC（Table of Contents，目录表） 是一个 Python 对象的序列化形式，位于 TOC 偏移指定的位置，一直延伸到文件末尾

TOC 描述了每个模块的名称、存储位置、长度以及是否为包

  • ispkg：表示模块是否为一个包（package），1 则为包

  • pos：该模块的压缩数据在 PYZ 文件中的起始偏移

  • length：压缩数据占用的字节数

编写脚本通过读取 TOC 获得所有模块的压缩数据所在位置，然后逐个提取

这里报错是因为 Python 导入模块时会优先搜索当前工作目录

它实现了程序/函数的上传下载、内存任意读写、代码执行，以及一个基于文本模式匹配的远程许可证校验

重点是下面三个函数

  • 功能：将字符串 s 的每个字符转换为 8 位二进制字符串（带前导零），并全部拼接

  • 示例：s = "AB" → ord('A') = 65 → "01000001"，ord('B') = 66 → "01000010"，结果 "0100000101000010"

  • 用途：将变换后的中间密钥字符串展开为一串 '0'/'1' 字符，作为控制空格替换的掩码

  • 功能：将字符串 string 位置 p 的字符替换为 c，返回新字符串

  • 注意：Python 字符串不可变，所以采用列表转换方式实现单字符替换

  • 用途：在 License 文本的指定空格位置，将空格改为制表符 \t
